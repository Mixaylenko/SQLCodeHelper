SQL Code Helper — Транслятор матричных выражений в SQL
Программа SQL Code Helper предназначена для автоматической генерации SQL-запросов на основе матричных уравнений, записанных в расширенной многомерно-матричной алгебре. Она позволяет описывать сложные многотабличные соединения и агрегации в компактной форме, а затем преобразовывать их в готовый SQL-код для выполнения в реляционных СУБД (MS SQL Server и другие).

📌 Основные возможности
Автономный режим — ввод структуры таблиц и матричного уравнения через интерфейс, получение готового SQL-скрипта без подключения к серверу.
Режим подключения к MS SQL Server — прямое выполнение сгенерированных запросов, просмотр результатов, визуализация промежуточных таблиц.
Поддержка всех основных типов соединений и объединение:
UNION ALL (х, +) 
LEFT JOIN (х/, +/)
RIGHT JOIN (/х, /+)
INNER JOIN (/х/, /+/)
(вместо х звёздочка - просто в Readme она не ставится)

Модификаторы для точной настройки:
1) для JOIN:
L/key или L/left=right — соединение по ключам для ON (исключаются из результата, участвуют в свёртке)
L/key или L/left=right — скоттовы ключи (остаются в результате, участвуют в соединении)
2) для UNION ALL
p/left=right — для указания разноимённых ключей соответствие при UNION ALL (колическтво ключей должно быть одинаково)
3) общий
m/key;m/left=right — исключение столбца из результата

Модификаторы записываются перед скобками с действием:
L/key(уравнение)

left=right - это такие комбинации как:
key1=key2
table1.key1=key2
key1=table2.key2
table1.key=table2.key

При необходимости нескольких соответствий и модификаторов запишите их через запятую:
L/key1,key2,m/key3(уравнение)

операнды записываются как table.key1,...,keyi

Пример:
L/ID,l/Region(Product.Price*/Discount.DiscountRate)


Если нужно сменить ключ перед новым действием, то предыдущий блок возьмите в скобки, поставьте точку и укажите новый ключ:
(блок1).NewKey блок2

Пример:
L/ID,l/Region,Category(Product.Price*/Discount.DiscountRate).Cost*/DiscountMaterial.DiscountRate

для проверки данные таблицы (для примера в автономном режиме - 2 вкладка):
Product/ID,Region,Category,Name,Price,Cost,Weight
Discount/ID,Region,Category,DiscountRate,Date
DiscountMaterial/ID,Region,Category,DiscountRate,Date

Плюсы:
Стабильность создания вичислительных запросов с получением предполагаемых результатов при любой логики уравнения в рамках допустимых конструкций.
Обработка неопределённых значений (NULL где нужно) — автоматическая подстановка нейтральных элементов (0 для сложения, 1 для умножения).
Кэширование промежуточных запросов — повторное использование подвыражений без дублирования кода.
Генерация именованных подзапросов (UA, IJ, LJ, RJ) для удобства отладки.
